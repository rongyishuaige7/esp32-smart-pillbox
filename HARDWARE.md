# 硬件与接线边界

本文只按当前公开源码记录接口和风险提示；它不是 PCB 原理图、电气审查结论、医疗建议或当前实物复测记录。

## 源码接口表

| 模块/信号 | ESP32 接口 | 当前源码证据 | 接线与边界 |
| :-- | :-- | :-- | :-- |
| 蜂鸣器 | GPIO18 | `firmware/src/main.cpp`、`reminder.cpp` | 只适用于低功率蜂鸣器/适当驱动；不可直接驱动扬声器、电机、继电器或市电负载。 |
| RGB LED | R=GPIO13、G=GPIO21、B=GPIO22 | `main.cpp`、`reminder.cpp` | 默认按共阳、低电平点亮；共阴模块需启用 `RGB_ACTIVE_HIGH=1` 后重新构建。每路需要合适限流。 |
| PIR 药仓 1 | GPIO15 | `sensorManager.begin(15, ...)` | 模块输出电平、供电、保持时间和实际安装位置尚未重新确认。 |
| PIR 药仓 2 | GPIO25（`INPUT_PULLDOWN`） | `sensorManager.begin(..., 25, ...)` | 输入逻辑、电平和实际抗干扰能力尚未重新确认。 |
| HX711 药仓 1 | DT=GPIO4、SCK=GPIO2 | `sensorManager.begin(..., 4, 2, ...)` | HX711、称重传感器供电、量程、标定和抗漂移均待当前实物复测。 |
| HX711 药仓 2 | DT=GPIO33、SCK=GPIO32 | `sensorManager.begin(..., 33, 32)` | 同上；不要把“计数阈值”当作精确剂量或医疗计量结果。 |
| SSD1306 OLED（可选代码路径） | SDA=GPIO26、SCL=GPIO27 | `display.cpp` | `displayBegin()` / `displayTick()` 当前没有从 `main.cpp` 调用；不要声称 OLED 已在本提交启用或验证。 |
| USB 串口 | 取决于具体 ESP32 开发板 | PlatformIO 上传/监视 | 开发板型号、Flash、USB 芯片、供电和实际烧录方式均待确认。 |

## 电源与安全注意事项

- ESP32 逻辑电平为 3.3 V，不能把 5 V 信号直接接到 GPIO；
- Wi-Fi、两个 HX711、RGB 和蜂鸣器可能引入瞬态电流与噪声，应使用稳定供电并确保所有低压模块共地；
- PIR、HX711、OLED、RGB、蜂鸣器和 ESP32 的精确模块型号、供电范围、上拉/下拉、限流与接线方向必须以实物丝印和资料为准；
- 本项目不包含电池、充电、药品保存、儿童防护、外壳、消防、故障安全、剂量校准或任何医疗认证设计；
- 服药提醒和重量/人体感应的源码逻辑只能用于受控教学实验，不能作为实际服药、漏服、诊断或安全判断的唯一依据。
