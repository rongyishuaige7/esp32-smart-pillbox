# 安全边界与漏洞报告

这是一个用于学习与原型复现的 ESP32 智能药盒历史项目，不是医疗器械、紧急呼叫设备、诊断工具或安全认证系统。它不应被用于代替医嘱、护理流程、人工确认或紧急处置。

## 当前网络边界

- 固件提供 AP 配网与局域网 REST；HTTP 没有 TLS、认证、会话、授权或请求签名；
- 源码中的 `SmartPillbox` / `12345678` 是公开教学 AP 配置，不能作为个人、家庭、学校或企业网络的安全凭据；
- 只在隔离、可信的本地测试网络中运行，禁止直接暴露到公网；
- `/config` 会处理 Wi-Fi 配置。不要把真实 SSID、密码、设备 IP、串口日志或 SPIFFS 导出提交到仓库、Issue、截图或视频；
- 应用端的“本次状态请求成功”仅表示当前 HTTP 请求得到预期响应，不是远端在线、提醒送达、服药完成或硬件健康的证明。

## 当前真机边界

当前提交没有重新烧录到真实设备，也没有重新验证称重、PIR、RGB、蜂鸣器、SPIFFS、Wi-Fi、NTP、AP 配网、REST 或 Flutter 端到端链路。请先阅读 [docs/PROJECT_STATUS.md](docs/PROJECT_STATUS.md) 与 [docs/VERIFICATION.md](docs/VERIFICATION.md)。

## 报告方式

请不要在公开 Issue 中粘贴密码、密钥、私网地址、个人健康数据、可识别身份信息或可复用的攻击细节。请通过 GitHub 账户主页的公开联系方式进行最小化说明；在收到确认前，不要公开可能影响其他使用者的细节。
